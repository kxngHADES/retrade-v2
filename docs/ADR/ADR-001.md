# ADR-001: Sending SMS via python without twilio

## Context
We knew that there were other local cheaper methods for sending SMS to users without the need of big brands like twilio

I was uncertain if there is a South African solution i could use

## Decision
I am using **WinSMS** because i found it was the easiest to use since all i do is use the normal email client logic BUT i use the clients phone number instead of they email address


## Consequences
- Positive:
	 - Cheaper
	 - Local
	 - can pick an exact amount if credit needed
	 - bulk credits are cheaper
- Negative
	 - purchase of credits takes time since it must be processed during working hours
	 - Credits could run out while at night so not working hours
- Trade-offs accepted
	 - we will not have 24/7 credit
